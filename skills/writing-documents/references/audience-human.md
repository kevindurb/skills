# Writing for Humans

Humans are not slow agents. They process information fundamentally differently.

## Core Principles

### Attention Economy

Human attention is scarce and competitive. Every element must earn its place or be cut.

- Attention is decided in seconds, not earned gradually
- First impression determines investment
- Competing with everything else demanding their time

### Motivation First

Humans need to know why they should care before investing attention.

- What problem does this solve for them?
- What happens if they ignore this?
- Why now rather than later?

### Scanning Behavior

Humans scan before reading. Structure documents for non-linear consumption.

- Headers are navigation, not decoration
- First sentence of each section carries disproportionate weight
- Bold key terms for scanning eyes
- Table of contents for longer documents

### Cognitive Load

Working memory holds roughly seven items. Exceed this and comprehension collapses.

- Group related items (3-5 per group)
- Introduce concepts before using them
- One new idea per paragraph
- Whitespace is cognitive breathing room

### Trust Signals

Humans need credibility signals before accepting information.

- Links to sources (even if unclicked)
- Acknowledgment of limitations
- Professional presentation
- Consistency with their existing knowledge

### Visual Processing

Humans extract meaning from visual structure faster than from linear text.

- Diagrams for relationships and flows
- Tables for comparisons
- Formatting (bold, bullets) for emphasis
- Whitespace for separation

## Document Structure for Humans

### Three-Segment Pattern

**Beginning (Hook + Gestalt)**
- Must catch attention AND convey core understanding in first screen
- Key points front-loaded
- Tell them what problem this solves

**Middle (Depth)**
- For those who want more
- Structure for scanning—not everyone reads this
- Progressive complexity

**End (Proof)**
- For skeptics
- Show mental working
- Reinforce key takeaways

### Repetition as Reinforcement

Unlike agents, humans benefit from encountering key ideas multiple times.

- Summary at top (what you'll learn)
- Body with full explanation
- Conclusion reinforcing main points
- Same idea, different angles—not redundancy

## Avoid "Claudeisms"

Certain patterns have become associated with unreviewed AI output and trigger distrust:

| Pattern | Problem |
|---------|---------|
| "It's not just X, it's Y!" | Performative reframing |
| "Key insight:" / "Critical:" | Telling readers what to find important |
| Pithy italicized closers | Fortune cookie energy |
| Excessive em-dashes | Prose feels AI-generated |
| "Let's now turn to..." | Formal transitions that add nothing |
| "It's worth noting..." | Hedging that delays the point |

The patterns aren't inherently wrong—they've become triggers for readers who spot content the sender didn't care enough to review.

**Exception**: "If X, then Y" closers are actionable tests, not decoration.

---

## What Hinders Humans

| Pattern | Problem |
|---------|---------|
| Wall of text | Decides not worth the effort |
| Buried lead | Never finds the point |
| No clear benefit | Why should I care? |
| Jargon without context | Feels exclusionary |
| Dense opening | Bounces immediately |
| No visual breaks | Exhausting to read |

---

## Checklist

- [ ] Hook in first sentence establishes relevance
- [ ] Scannable structure with meaningful headers
- [ ] Key terms bolded for scanning
- [ ] Cognitive load managed (grouped items, whitespace)
- [ ] Trust signals present (sources, limitations acknowledged)
- [ ] Visual elements aid comprehension
- [ ] No Claudeisms that trigger distrust
- [ ] Reinforcement through varied repetition
- [ ] Clear benefit stated early
